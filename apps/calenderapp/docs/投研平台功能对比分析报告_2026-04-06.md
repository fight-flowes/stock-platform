## 1. 现有平台功能梳理

### 1.1 当前已实现功能

| 功能模块 | 具体功能 | 完成度 |
|---------|---------|--------|
| **交易日历** | 交易日/休息日标记、农历日期、节假日管理 | ✅ 完成 |
| **事件管理** | 投研事件记录、重要性分级、关联股票、日历视图展示 | ✅ 完成 |
| **涨停股分析** | 涨停列表、连板榜、龙头识别 | ✅ 完成 |
| **资金流向** | 机构/游资/北向净买入排行、龙虎榜数据 | ✅ 完成 |
| **概念热度** | 概念出现频率统计、热门概念板块 | ✅ 基础完成 |
| **深度分析** | 单股涨停分析报告（AI 分析）、强度评分 | ✅ 完成 |
| **K线展示** | 近60日K线、当日分时线 | ✅ 完成 |
| **数据同步** | Tushare 数据同步、批量同步、分区表架构 | ✅ 完成 |
| **股票管理** | 股票基础信息、交易所分类 | ✅ 完成 |
| **用户体验** | 亮色/暗色主题、Token认证、分页筛选 | ✅ 完成 |

### 1.2 现有平台优势

1. **数据架构优秀**：分区表设计解决数据膨胀，支持按月分区管理
2. **自动化程度高**：Tushare 自动同步、龙头自动识别、AI 深度分析
3. **分析维度丰富**：涨停强度评分、资金特征分析、涨停原因分类
4. **报告集成能力**：支持接入外部产出的 Markdown 分析报告并统一展示

### 1.3 现有平台短板

1. **板块分析薄弱**：仅有概念热度统计，缺乏板块轮动追踪
2. **缺少预警系统**：无涨停预警、无板块异动通知
3. **数据展示单一**：缺乏可视化图表（如板块热力图、资金流向图）
4. **历史对比不足**：缺少与历史同类型涨停的对比分析
5. **自选股缺失**：无个人自选股管理、无个性化关注列表

## 4. 功能新增建议（按优先级排序）

### 4.1 P0 - 核心功能（立即新增）

#### 4.1.1 涨停复盘报告生成

**功能描述**：每日收盘后自动生成涨停复盘报告，总结当日涨停情况、热点板块、龙头股表现。

**具体内容**：
- 当日涨停概况（涨停数量、炸板数量、连板股数量）
- 热点板块 TOP5（涨幅、涨停股数量、龙头股）
- 龙头股表现（总龙头、板块龙头、连板高标）
- 资金动向总结（机构/游资/北向净买入 TOP 股票）
- 概念题材热度排行
- 次日关注点（潜在接力标的、新热点预判）

**数据来源**：
- 现有涨停数据 + Tushare 板块数据

**实现方案**：
- 新增 `/api/limit-up/daily-review` 接口
- 新增 `DailyReviewService` 服务
- 前端新增"每日复盘" Tab 页

**预计工作量**：2-3 天

---

#### 4.1.2 板块热度分析模块

**功能描述**：实时展示板块涨跌幅排行、板块资金流向、板块涨停股分布。

**具体内容**：
- 板块涨跌幅排行（行业板块、概念板块）
- 板块资金流向（板块内资金净流入/流出）
- 板块涨停股统计（每个板块涨停股数量、龙头股）
- 板块热力图（可视化展示板块热度）

**数据来源**：
- Tushare `index_classify`（行业分类）
- Tushare `concept`（概念板块）
- Tushare `moneyflow`（板块资金流向）

**实现方案**：
- 新增 `板块数据表`（板块基础信息）
- 新增 `/api/sector/` 接口
- 新增 `SectorService` 服务
- 前端新增"板块热度" Tab 页，使用热力图组件

**预计工作量**：3-4 天

---

#### 4.1.3 板块轮动追踪

**功能描述**：记录板块轮动规律，分析板块启动-发酵-高潮-退潮周期。

**具体内容**：
- 板块生命周期追踪（启动日、高潮日、退潮日）
- 板块轮动规律统计（板块切换周期、领涨板块轮换）
- 板块强度评分（板块内涨停股数量、龙头高度、持续性）
- 历史板块轮动复盘（历史类似行情的板块轮动规律）

**数据来源**：
- 历史涨停数据 + 板块数据 + 人工标注

**实现方案**：
- 新增 `板块状态表`（板块每日状态：启动/发酵/高潮/退潮）
- 新增 `板块轮动分析` 服务
- 前端新增"板块轮动"分析页面

## 5. 技术实现方案

### 5.1 数据库扩展

#### 5.1.1 新增表结构

```sql
-- 板块基础信息表
CREATE TABLE sc.sectors (
    id SERIAL PRIMARY KEY,
    sector_code VARCHAR(32) UNIQUE,  -- 板块代码
    sector_name VARCHAR(128),        -- 板块名称
    sector_type VARCHAR(32),         -- 板块类型: industry/concept
    parent_sector VARCHAR(64),       -- 父板块
    created_at TIMESTAMPTZ,
    updated_at TIMESTAMPTZ
);

-- 板块每日状态表
CREATE TABLE sc.sector_daily_stats (
    id SERIAL PRIMARY KEY,
    sector_code VARCHAR(32),
    trade_date DATE,
    pct_chg DOUBLE,                  -- 板块涨跌幅
    limit_up_count INTEGER,          -- 涨停股数量
    dragon_head_code VARCHAR(32),    -- 板块龙头
    dragon_height INTEGER,           -- 龙头高度（连板数）
    fund_net_inflow DOUBLE,          -- 资金净流入
    sector_status VARCHAR(32),       -- 板块状态: startup/rising/peak/falling
    created_at TIMESTAMPTZ
);

-- 自选股表
CREATE TABLE sc.watchlist (
    id SERIAL PRIMARY KEY,
    user_id INTEGER,                 -- 用户ID（预留）
    stock_code VARCHAR(32),
    group_name VARCHAR(64),          -- 分组名称
    notes TEXT,                      -- 备注
    created_at TIMESTAMPTZ,
    updated_at TIMESTAMPTZ
);

-- 预警规则表
CREATE TABLE sc.alert_rules (
    id SERIAL PRIMARY KEY,
```
