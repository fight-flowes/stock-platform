## 一、功能概述

### 1.1 目标

每日收盘后自动生成涨停复盘报告，快速了解当日涨停全景，包括：
- 市场概况（涨停数量、封板率、情绪判断）
- 热点板块 TOP5
- 龙头股表现
- 资金动向总结
- 次日关注点建议

### 1.2 报告示例结构

```markdown

- 连板股：12 只（2板 8只，3板 3只，4板+ 1只）
- 封板率：84%
- 市场情绪：🔥 活跃

## 二、热点板块 TOP5
| 排名 | 板块 | 涨停股 | 龙头 | 龙头高度 |
|------|------|--------|------|----------|
| 1 | 低空经济 | 8只 | 中超控股 | 4板 |
| 2 | 机器人 | 5只 | xxx | 2板 |

## 三、龙头股表现
- 🏆 总龙头：中超控股（4板，强度★★★★★）
- 🎯 板块龙头：低空经济-中超控股、机器人-xxx
```

## 三、数据库设计

### 3.1 新增表：`daily_reviews`

```sql
CREATE TABLE sc.daily_reviews (
    id SERIAL PRIMARY KEY,
    review_date DATE UNIQUE NOT NULL,
    
    -- 市场概况
    limit_up_count INTEGER DEFAULT 0,        -- 涨停总数
    sealed_count INTEGER DEFAULT 0,          -- 封住涨停
    broken_count INTEGER DEFAULT 0,          -- 炸板数量
    consecutive_count INTEGER DEFAULT 0,     -- 连板股数量
    seal_rate INTEGER DEFAULT 0,             -- 封板率(%)
    market_sentiment VARCHAR(32),            -- hot/active/normal/weak
    
    -- 热点板块（JSONB 数组）
    hot_sectors JSONB,
    -- 格式: [{"rank":1,"sector":"低空经济","limit_up_count":8,"dragon":"中超控股","dragon_height":4}]
    
    -- 龙头股
    total_dragon JSONB,                      -- 总龙头信息
    sector_dragons JSONB,                    -- 板块龙头列表
    
    -- 资金动向
    institution_top JSONB,                   -- 机构净买入TOP
    hot_money_top JSONB,                     -- 游资净买入TOP
    
    -- 概念热度
    hot_concepts JSONB,                      -- 热门概念
    
    -- 次日关注
    watch_stocks JSONB,                      -- 潜在接力标的
    new_sectors JSONB,                       -- 新启动板块
    risk_alerts JSONB,                       -- 风险提示
    
    -- 完整报告（Markdown）
    full_report TEXT,
    
    -- 元数据
    auto_generated BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_daily_reviews_date ON sc.daily_reviews(review_date);
```

```python
class DailyReviewService:
    
    @classmethod
    def generate_review(cls, review_date: date) -> Dict:
        """生成复盘报告"""
        
        # 1. 收集市场概况数据
        market_data = cls._collect_market_data(review_date)
        # 查询 limit_up_stocks 表
        # - COUNT(*) 涨停总数
        # - COUNT WHERE limit_up_type IN ('first_board', 'multi_board') 封住涨停
        # - COUNT WHERE limit_up_type = 'broken_board' 炸板
        # - COUNT WHERE consecutive_days >= 2 连板股
        
        # 2. 收集板块热度数据
        sector_data = cls._collect_sector_data(review_date)
        # GROUP BY industry, 统计每个板块涨停股数量
        # 找到每个板块的龙头（MAX consecutive_days + MAX strength_score）
        
        # 3. 收集龙头股数据
        dragon_data = cls._collect_dragon_data(review_date)
        # 总龙头：is_dragon_head=True, MAX consecutive_days
        # 板块龙头：is_dragon_head=True, 按 industry 分组
        
        # 4. 收集资金动向数据
        fund_data = cls._collect_fund_data(review_date)
        # ORDER BY institution_net_buy DESC LIMIT 10
        # ORDER BY hot_money_net_buy DESC LIMIT 10
        
        # 5. 收集概念热度数据
        concept_data = cls._collect_concept_data(review_date)
        # 展开 concept_tags JSONB 数组，统计概念出现次数
        
        # 6. 计算市场情绪
        sentiment = cls._calculate_sentiment(market_data)
        # 规则：
        # - hot: 涨停数 > 50 且 封板率 > 80%
        # - active: 涨停数 > 30 或 封板率 > 70%
        # - normal: 涨停数 15-30
        # - weak: 涨停数 < 15

        # 潜在接力：连板股 + 强度评分 >= 4 + 龙头身份
        # 新启动板块：涨停股 >= 3 且 龙头高度 <= 2
        # 风险提示：板块涨停 >= 5 且 龙头高度 >= 4
        
        # 8. 组装完整报告（Markdown）
        full_report = cls._generate_full_report(...)
        
        # 9. 存入数据库
        review = DailyReview(...)
        session.add(review)
        
        return review.to_dict()
```

## 六、核心算法

### 6.1 市场情绪判断

```python
def calculate_sentiment(market_data):
    """
    情绪判断规则：
    - hot: 涨停数 > 50 且 封板率 > 80%
    - active: 涨停数 > 30 或 封板率 > 70%
    - normal: 涨停数 15-30
    - weak: 涨停数 < 15 或 封板率 < 50%
    """
    limit_up_count = market_data["limit_up_count"]
    seal_rate = market_data["seal_rate"]
    
    if limit_up_count > 50 and seal_rate > 80:
        return "hot"
    elif limit_up_count > 30 or seal_rate > 70:
        return "active"
    elif limit_up_count >= 15:
        return "normal"
    else:
        return "weak"
```

### 6.2 次日关注点生成

```python
def generate_watch_points(dragon_data, sector_data):
    """
    潜在接力标的：
    - 总龙头：height >= 3 且 strength >= 4
    - 板块龙头：height >= 2 且 strength >= 3
    
    新启动板块：
    - 涨停股 >= 3 且 龙头高度 <= 2
    
    风险提示：
    - 板块涨停 >= 5 且 龙头高度 >= 4 → 板块高潮，注意分化
    """
    watch_stocks = []
    new_sectors = []
    risk_alerts = []
    
    # 总龙头接力
    if dragon_data["total_dragon"]["height"] >= 3:
        watch_stocks.append(dragon_data["total_dragon"])
    
    # 板块龙头接力
    for dragon in dragon_data["sector_dragons"]:
        if dragon["height"] >= 2:
            watch_stocks.append(dragon)
    
    # 新启动板块
    for sector in sector_data["hot_sectors"]:
        if sector["limit_up_count"] >= 3 and sector["dragon_height"] <= 2:
            new_sectors.append(sector)
    
    # 风险提示
    for sector in sector_data["hot_sectors"]:
        if sector["limit_up_count"] >= 5 and sector["dragon_height"] >= 4:
            risk_alerts.append({
                "sector": sector["sector"],
                "reason": "板块高潮，注意分化"
            })
    
    return watch_stocks, new_sectors, risk_alerts
```
