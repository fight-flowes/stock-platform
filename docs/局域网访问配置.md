# stock-platform 局域网访问配置

本文记录如何让 `stock-platform` 在同一局域网内通过 Windows 主机 IP 访问。

适用环境：

- Windows 主机 IP：`10.168.4.11`
- WSL IP：`192.168.128.173`
- 前端对外端口：`13000`
- 后端对外端口：`15000`
- stockkb 对外端口：`18040`

## 1. 启动方式

本机开发：

```bash
cd /home/leisaihua/workspace/stock_info/stock-platform
bash scripts/dev-up.sh
```

局域网访问：

```bash
cd /home/leisaihua/workspace/stock_info/stock-platform
bash scripts/dev-up-lan.sh
```

说明：

- `dev-up.sh` 适合本机访问，前端默认请求 `http://127.0.0.1:5000`
- `dev-up-lan.sh` 适合局域网访问，前端默认请求 `http://10.168.4.11:15000`
- 不要同时启动两个脚本，它们会争用同一组服务端口

## 2. Windows 端口转发

在 Windows PowerShell 中执行。

先删除旧规则；如果提示“系统找不到指定的文件”，可忽略。

```powershell
netsh interface portproxy delete v4tov4 listenport=13000 listenaddress=0.0.0.0
netsh interface portproxy delete v4tov4 listenport=15000 listenaddress=0.0.0.0
netsh interface portproxy delete v4tov4 listenport=18040 listenaddress=0.0.0.0
```

添加转发规则：

```powershell
netsh interface portproxy add v4tov4 listenport=13000 listenaddress=0.0.0.0 connectport=3000 connectaddress=192.168.128.173
netsh interface portproxy add v4tov4 listenport=15000 listenaddress=0.0.0.0 connectport=5000 connectaddress=192.168.128.173
netsh interface portproxy add v4tov4 listenport=18040 listenaddress=0.0.0.0 connectport=8040 connectaddress=192.168.128.173
```

## 3. Windows 防火墙放行

```powershell
New-NetFirewallRule -DisplayName "WSL2-stock-platform-13000" -Direction Inbound -Protocol TCP -LocalPort 13000 -Action Allow
New-NetFirewallRule -DisplayName "WSL2-stock-platform-15000" -Direction Inbound -Protocol TCP -LocalPort 15000 -Action Allow
New-NetFirewallRule -DisplayName "WSL2-stock-platform-18040" -Direction Inbound -Protocol TCP -LocalPort 18040 -Action Allow
```

## 4. 检查命令

查看转发规则：

```powershell
netsh interface portproxy show all
```

查看 Windows 端口是否监听：

```powershell
netstat -an | findstr 13000
netstat -an | findstr 15000
netstat -an | findstr 18040
```

查看 WSL 侧服务状态：

```bash
cd /home/leisaihua/workspace/stock_info/stock-platform
bash scripts/dev-status.sh
```

## 5. 访问地址

本机访问：

- 前端：`http://127.0.0.1:3000`
- 后端：`http://127.0.0.1:5000`
- stockkb：`http://127.0.0.1:8040`

局域网访问：

- 前端：`http://10.168.4.11:13000`
- 后端：`http://10.168.4.11:15000`
- stockkb：`http://10.168.4.11:18040`

## 6. 常见问题

### 访问前端显示“加载失败”

通常是后端没有启动，或者前端请求的 API 地址不对。

先检查：

```bash
bash scripts/dev-status.sh
```

如果只是本机访问，优先使用：

```bash
bash scripts/dev-up.sh
```

如果是局域网访问，优先使用：

```bash
bash scripts/dev-up-lan.sh
```

### `stockkb` 启动失败，提示端口绑定失败

说明 `8040` 已被其他进程占用。先停止旧进程，或排查已有实例，再重新执行启动脚本。

### WSL 重启后局域网访问失效

WSL IP 可能变化了。需要重新确认新的 WSL IP，并更新 Windows 的 `portproxy` 规则里的 `connectaddress`。
